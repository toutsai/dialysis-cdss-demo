# Dialysis CDSS 主頁面導覽手冊

版本：2026-05-04 demo / 測試版

這份文件用來協助同事第一次使用 `Dialysis CDSS` 查房工作台。主頁面分成三個大欄位：左側系統列、中間床位總表與提醒、右側病人詳情。建議使用流程是：先登入，從中間選擇班別與病人，再到右側查看或更新病人資料。

> 注意：目前 demo 版資料為測試資料或模擬資料，不可直接用於正式臨床決策。正式使用前仍需確認資料來源、院內規則、權限與通知流程。

## 一、左側系統列

左側淡藍色區塊是系統導覽與登入區。

### 1. 目前使用者與角色

登入後會顯示目前使用者與角色，例如：

- 管理員
- 醫師
- 護理長
- 護理師

不同角色可編輯的範圍不同。沒有編輯權限的頁面仍可查看，但不能修改。

### 2. 頁面切換

左側 `頁面` 可切換不同功能：

- `查房工作台`：主要工作畫面，查看床位、提醒與病人詳情。
- `病人清單`：維護目前洗腎室病人、床位、頻率、班別與轉出資訊。
- `人員設定`：維護使用者、角色、帳號與密碼。
- `本院藥物清單`：維護洗腎相關藥物品項，供洗腎藥物頁面下拉選單使用。
- `規則設定`：調整 ESA、鐵劑、CKD-MBD 等建議規則門檻。

### 3. 模糊病人資訊

勾選 `模糊病人資訊` 後，畫面會以遮罩方式顯示病人姓名與病歷號，例如 `黃O全` 或部分隱藏病歷號。這只影響畫面顯示，資料庫仍保留原始資料。

### 4. 重新同步 CSV

`重新同步 CSV` 用於重新讀取本機 CSV / 匯入資料並同步到 SQLite。未來若由院內系統定期匯出抽血資料，也會走類似的同步概念。

## 二、中間床位總表與提醒區

中間欄位是查房時的「病人選擇區」。目標是快速找到今天要看的病人。

### 1. 床位總表人數

標題會顯示目前名單總人數，例如 `床位總表 | 12 人`。目前床位總表只顯示尚未轉出的病人。

### 2. 頻率與班別篩選

可直接點選頻率與班別，不需要下拉選單。

- 頻率：例如 `一三五`、`一五`、`二四六`
- 班別：例如 `午班`、`早班`

頻率與班別會依現有病人資料動態產生。若新增病人使用新的頻率或班別，查房工作台會自動出現該選項。

### 3. 搜尋

搜尋框可用來搜尋：

- 姓名
- 病歷號
- 床號

適合在病人數增加後快速定位病人。

### 4. 目前篩選摘要

篩選後會顯示目前條件與人數，例如：

`一三五 > 午班 | 3 人`

這代表目前清單是 `一三五` 頻率、`午班` 的 3 位病人。

### 5. 今日提醒

`今日提醒` 目前主要顯示到期且尚未處理的醫護交班。

提醒卡片顯示床號與病人姓名。點選提醒卡片後，系統會自動切換到該病人的醫護交班紀錄，方便直接處理。

### 6. 病人列表

病人列表顯示目前篩選條件下的病人，格式簡化為：

`床號 病人姓名`

點選病人後，右側會顯示該病人的詳情。被選取的病人會以較明顯底色標示。

## 三、右側病人詳情區

右側是主要工作區。上方先顯示病人摘要，下方用頁籤切換不同資料。

### 1. 病人摘要列

最上方顯示：

- 病人姓名
- 病歷號或識別碼
- DW
- AK
- 藥水 Ca
- 透析頻率
- 班別
- 床位

這區讓查房者不用進入表格，就能先看到最重要的透析資訊。

### 2. 頁籤總覽

右側目前包含 6 個頁籤：

- `主要問題`
- `近期事件`
- `醫護交班`
- `透析醫囑`
- `洗腎藥物`
- `治療趨勢與調整建議`

前 5 個頁籤採相同操作邏輯：上方是可展開的新增區塊，下方是既有或歷史累積紀錄。預設會先收合新增區塊，方便使用者先閱讀紀錄。

## 四、各頁籤功能

### 1. 主要問題

用途：記錄病人的長期疾病背景與目前待處理問題。

新增主要問題時可選擇問題類別：

- `Underlying disease`
- `現在待處理問題`

同一筆問題可以同時屬於兩個類別。下方會分成兩段顯示：

- `Underlying disease`
- `現在待處理問題`

可記錄欄位包含主要問題、狀態、負責角色、問題類別、更新者、更新時間與備註。新增或修改 `現在待處理問題` 時，可觸發外部通知設定，例如 LINE 或 email。

### 2. 近期事件

用途：記錄近期急診、住院、門診或重要臨床事件。

新增欄位包含：

- 事件日期
- 事件類型：急診、住院、門診
- 事件標題：血管通路問題、心臟問題、感染問題、周邊血管問題、呼吸衰竭、其他
- 事件內容

下方會累積既有近期事件，方便查房醫師快速掌握病人最近狀況。

### 3. 醫護交班

用途：建立針對特定病人與目標日期的交班提醒。

新增欄位包含：

- 目標日期
- 交班類型：共同交班、醫師交班、護理交班
- 優先度
- 事件標題
- 交班內容

若目標日期到期且狀態仍未完成，會出現在中間欄位的 `今日提醒`。點選提醒卡片可直接跳到該病人的交班紀錄。

### 4. 透析醫囑

用途：記錄或調整透析處方。

新增 / 調整欄位包含：

- 生效日期
- 透析日：一到六用點選方式選擇，系統會自動解讀為頻率
- 班別：早班、午班、晚班
- 床位
- AK
- 藥水 Ca
- Blood flow
- Dialysate flow
- Dry weight
- 抗凝 Loading
- 抗凝 Maintain
- 血管通路側別與類型：左 / 右、FVC / PERM / AVF / AVG
- 備註

下方 `既有透析醫囑` 會保留歷史紀錄，用來追蹤不同時間點的處方變化。

### 5. 洗腎藥物

用途：新增或調整透析相關藥物，並追蹤藥物變化。

新增 / 調整欄位包含：

- 調整 / 新增日期
- 類別
- 品項
- 劑量
- 單位
- 頻率
- 狀態
- 醫囑 / 藥品碼
- 備註

類別與品項來自左側 `本院藥物清單` 的設定。點選下方既有藥物矩陣中的一列，可自動帶入既有品項、劑量與頻率，儲存後會新增一筆調整紀錄，不會覆蓋舊資料。

下方 `既有洗腎藥物` 目前以調整日期作為欄位，藥物作為列，中間顯示當時的劑量與頻率，方便比較治療介入的時間序列。

### 6. 治療趨勢與調整建議

用途：整合抽血趨勢、洗腎藥物、透析醫囑與規則引擎建議。

此頁面分成幾個層次：

- 重點摘要：顯示 Hb、P、cCa、iPTH、Kt/V 等目前狀態與趨勢。
- 累積趨勢表：以月份為欄位、檢驗項目為列，異常值會用底色標示。
- 治療介入時間軸：顯示日期、藥名或醫囑項目、變動內容與更新者。
- 建議區塊：依貧血 / ESA / 鐵劑、CKD-MBD、透析充分性與醫囑分組呈現。

若醫師同意建議，可建立藥物或醫囑調整草稿，再回到 `洗腎藥物` 或 `透析醫囑` 確認並正式儲存。

## 六、建議查房操作流程

1. 從左側登入，確認目前角色正確。
2. 進入 `查房工作台`。
3. 在中間欄選擇頻率與班別。
4. 查看 `今日提醒`，優先處理到期交班。
5. 從病人列表點選病人。
6. 右側先看病人摘要列，確認 DW、AK、藥水 Ca、班別與床位。
7. 依序查看主要問題、近期事件與醫護交班。
8. 需要調整時，再展開新增區塊新增資料。
9. 若要看月抽血與治療反應，切到 `治療趨勢與調整建議`。
10. 若同意建議，先建立草稿，再到醫囑或藥物頁籤確認後儲存。

## 七、測試版注意事項

- Demo 版可供同事操作流程，但不應放入真實病人資料。
- 若使用真實資料庫測試，建議開啟 `模糊病人資訊`。
- 抽血資料目前可由匯入檔案或模擬資料提供；正式串接前仍需資訊室確認資料格式。
- 藥物與醫囑目前以前台輸入為主，日後可再評估是否串接 HIS。
- 規則建議只作為輔助提醒，正式臨床處置仍需醫師判斷與簽核。
